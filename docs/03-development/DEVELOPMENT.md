# Development Setup

Use `xtask` as the contributor control panel. Run every command from the repository root.

## Quick Start

```bash
cargo xtask setup
cargo xtask report
cargo xtask test --list
cargo xtask check --list
```

## Complete xtask Command Reference

Arguments written as `<name>` are required; arguments written as `[name]` are optional. The
`test apps` commands are live workflows through `apps/cli` and the NAPI bridge, so several of
them read or modify the real profile store. The crate and sidecar lists are discovered from the
workspace at runtime.

| Area            | Command                                                         | Job                                                                                                       | Important behavior                                                                                                                                   |

| Help            | `cargo xtask --help`                                            | Lists top-level xtask commands.                                                                           | Read-only.                                                                                                                                           |
| Help            | `cargo xtask <command> --help`                                  | Shows parser-level arguments and flags for one command.                                                   | Read-only; nested `test` and `check` routes use their own `--list` commands below.                                                                   |
| Setup           | `cargo xtask setup`                                             | Checks Qt/CMake, Python, Rust/Cargo, and Node/NPM readiness.                                              | Safe mode: checks, user-level attempts, and remediation guidance.                                                                                    |
| Setup           | `cargo xtask setup --all`                                       | Checks every component and attempts broader installation.                                                 | May attempt admin-level OS package installation.                                                                                                     |
| Setup           | `cargo xtask setup --qt`                                        | Checks only Qt and CMake.                                                                                 | Multiple component flags can be combined.                                                                                                            |
| Setup           | `cargo xtask setup --py`                                        | Checks only Python.                                                                                       | Multiple component flags can be combined.                                                                                                            |
| Setup           | `cargo xtask setup --cargo`                                     | Checks only Rust and Cargo.                                                                               | Multiple component flags can be combined.                                                                                                            |
| Setup           | `cargo xtask setup --npm`                                       | Checks only Node.js and NPM.                                                                              | Multiple component flags can be combined.                                                                                                            |
| Report          | `cargo xtask report`                                            | Runs tool, Rust, renderer, sidecar-artifact, size, and version-consistency checks.                        | Informational: reports failures but exits successfully.                                                                                              |
| Report          | `cargo xtask report --strict`                                   | Runs the same health report in enforcement mode.                                                          | Returns an error if any report check fails.                                                                                                          |
| Build           | `cargo xtask build`                                             | Builds the default target set: OCR, STT, capture, and Tauri.                                              | Because Tauri is included, `COMMIT_SHA` or a positional commit SHA is required.                                                                      |
| Build           | `cargo xtask build all`                                         | Selects the default target set.                                                                           | Equivalent target selection to bare `build` and `build --all`.                                                                                       |
| Build           | `cargo xtask build --all`                                       | Selects the default target set through the Clap flag.                                                     | Exclusions may be added, for example `--all -ocr`.                                                                                                   |
| Build           | `cargo xtask build <target>...`                                 | Builds one or more explicit targets.                                                                      | Supported targets: `ocr`, `stt`, `capture`, `capture-qt`, `tauri`, `electron`, and `cli`.                                                            |
| Build           | `cargo xtask build -<target>`                                   | Builds the default set minus a target.                                                                    | Exclusions can be combined with `all`, `--all`, or explicit includes.                                                                                |
| Build           | `cargo xtask build ocr`                                         | Builds and packages the PaddleOCR sidecar.                                                                | Does not require a commit SHA.                                                                                                                       |
| Build           | `cargo xtask build stt`                                         | Builds the Whisper STT sidecar.                                                                           | Does not require a commit SHA.                                                                                                                       |
| Build           | `cargo xtask build capture`                                     | Builds the complete Qt capture engine and deployable artifacts.                                           | Used to populate shell sidecar binaries.                                                                                                             |
| Build           | `cargo xtask build capture-qt`                                  | Builds only the Qt portion of capture.                                                                    | Aliases: `captureqt` and `qt`.                                                                                                                       |
| Build           | `cargo xtask build tauri <COMMIT_SHA>`                          | Builds the archived Tauri desktop bundle for the host OS.                                                 | SHA must be 7, 8, or 40 hexadecimal characters; `COMMIT_SHA` is an alternative.                                                                      |
| Build           | `cargo xtask build electron <COMMIT_SHA>`                       | Selects the Electron build pipeline.                                                                      | Currently reserved and always fails because the pipeline is not implemented.                                                                         |
| Build           | `cargo xtask build cli`                                         | Selects the standalone CLI build target.                                                                  | Currently reserved and always fails; the live CLI harness is built by `test apps` instead.                                                           |
| Build           | `cargo xtask build ocr --measure-ocr-size`                      | Builds OCR and measures its packaged payload.                                                             | `SQUIGIT_OCR_MEASURE_SIZE=1` enables the same measurement.                                                                                           |

| Clean           | `cargo xtask clean`                                             | Removes generated OCR, STT, capture, shell binary, packaging, Cargo target, and archived Tauri artifacts. | Destructive to build caches and downloaded/generated sidecar assets; source files are untouched.                                                     |
| Run             | `cargo xtask dev --electron [args...]`                          | Starts renderer Vite on port 1420, builds the Electron main process, and launches Electron.               | Requires populated `apps/desktop/binaries`; cleans up Vite after Electron exits.                                                                     |
| Run             | `cargo xtask dev tray --electron [args...]`                     | Launches Electron through the dev path with tray mode selected.                                           | `tray` is the only meaningful mode token.                                                                                                            |
| Run             | `cargo xtask dev --tauri [args...]`                             | Runs the archived Tauri shell with its prebuilt renderer.                                                 | Requires populated archived Tauri binaries; bypasses Vite.                                                                                           |
| Run             | `cargo xtask dev tray --tauri [args...]`                        | Runs the archived Tauri shell in background/tray mode.                                                    | Passes `--background` to the shell.                                                                                                                  |
| Run             | `cargo xtask dev`                                               | Attempts to enter dev mode without choosing a shell.                                                      | Currently invalid: exactly one of `--electron` or `--tauri` is required.                                                                             |
| Run             | `cargo xtask run [cmd] [args...]`                               | Compatibility entry point for the archived Tauri shell.                                                   | Currently always executes `cargo run`; the accepted `[cmd]` value is not honored by the Tauri branch.                                                |

## Current Design Constraints Exposed by the Commands

- `test apps` mixes live/manual workflows with automated-test naming and always travels through
  the CLI/NAPI boundary.
- `build electron` and `build cli` are selectable but deliberately unimplemented.
- `run [cmd]` advertises arbitrary Tauri commands, but the current Tauri branch always performs
  `cargo run`.
- Bare `build` includes Tauri and therefore needs commit provenance even for a local default build.
- Versioning has no `--bump` mode; shell, OCR, and STT require explicit SemVer, while renderer and
  repository metadata generate CalVer automatically.

These constraints are documented as current behavior so they can be evaluated during the planned
xtask redesign rather than accidentally preserved.
